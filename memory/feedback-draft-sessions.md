Draft sessions must not include `Done` checkboxes or session log tables. These features signal a session is ready to use and encourage Ben to skip the required review step before each draft.

**Why:** Draft sessions are provisional plans that must be reviewed against the preceding session's log before use. Including checkboxes and a session log implies the session is finalised.

**How to apply:** When writing a draft session file, strip all `Done` columns from exercise tables and omit the session log section entirely. The draft callout block (`> **Draft session — review after Session N before doing this one.**`) is still required.
